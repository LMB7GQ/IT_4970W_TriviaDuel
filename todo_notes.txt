The "GameContext" (React Pro-Tip):
I moved all the messy socket logic into something called "React Context" (specifically GameContext.js). 
- Think of Context like a "Global Radio Station." Instead of passing data down manually through every single component (Prop-Drilling), the Context broadcasts the game state to the entire app. 
- Any page (Login, Playing, Results) can just "tune in" to the GameContext to get the current score, the socket connection, or the player's name. It keeps the code way cleaner and much easier to manage as the app grows.

Tips for the Demo:
- If you want to show the server is actually "alive," run the app in two different browser tabs and start a Ranked Match. It proves the backend is syncing everything in real-time.
- Stick to the mocked Solo/Bot modes if you haven't seeded the DB yet—they're "safe" and won't crash if the database is empty.
