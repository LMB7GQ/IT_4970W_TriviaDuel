What we fixed (The Connection):
Before this, the client and server were basically "broken" and not talking to each other at all. 
- Event Names: The server was shouting "matchReady" but the client was listening for "matchFound." They were totally out of sync.
- Data Mismatch: The server wanted an object with a player's rank, but the client was just sending a plain name string.
- Missing Phases: The server required a "Ban/Pick" phase that the client didn't even have a screen for yet. 
- File Casing: Some of the file imports on the server were using the wrong case (like Capital letters where they should've been lowercase), which would've crashed the whole thing on a Linux server.
Everything is perfectly aligned now, so they're finally speaking the same language.

The "GameContext" (React Pro-Tip):
I moved all the messy socket logic into something called "React Context" (specifically GameContext.js). 
- Think of Context like a "Global State Manager for the Client." Instead of passing data down manually through every single component (Prop-Drilling), the Context broadcasts the game state to the entire app. 
- Any page (Login, Playing, Results) can just "subscribe" or listen to the GameContext to get the current score, the socket connection, or the player's name. It keeps the code way cleaner and much easier to manage as the app grows.
